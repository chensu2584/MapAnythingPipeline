## 数据流总览

```
去畸变头图 (全分辨率)          views.npz (推理产物)
      │                        head_pts3d  H×W×3  每像素的世界坐标(米)
      ▼                        head_img    H×W×3  推理用的 518 集图
  [1] YOLO 推理                head_conf / head_mask
      │ 框 (+掩码)
      ▼
  [2] 掩码获取 (seg 直接出 / det 走 SAM2 或几何法)
      │ 二值掩码 @ 全分辨率
      ▼
  [3] 域与分辨率对齐 (remap → 腐蚀 → 最近邻缩放到 H×W)
      │ 二值掩码 @ pts3d 分辨率
      ▼
  [4] 3D 提升: obj_pts = head_pts3d[mask]     ← 核心,一行索引
      │ N×3 物体点云 (世界坐标)
      ▼
  [5] 清洗: DBSCAN 取最大簇
      ▼
  [6] 物体建档: 质心/AABB/OBB/主轴 → objects.json
      ▼
  [7] 写入体素栅格: 按分数加权投票 → 语义占据栅格
```

## [1] YOLO 推理:在哪张图上跑

推理必须在**全分辨率**头图上跑,而不是 `views.npz` 里那张 518 集的缩放图——缩放图是给 MapAnything 用的,小物体在上面可能只剩几个像素,YOLO 检测质量会明显下降。

具体跑在哪个"域"取决于 YOLO 训练数据(计划 §2.1 分支 B):

- **训练用去畸变图** → 直接在 `outputs/undistorted/<capture>/head.png` 上跑,后面零转换
- **训练用原始图** → 在原始 `head.png` 上跑(保持训练域,精度最优),然后做一次域转换,见 [3](a)

## [2] 掩码获取:框 ≠ 物体

3D 提升需要**像素级掩码**:框内必然混有背景像素,直接把框内所有像素提升到 3D,"物体点云"会包含物体后面的桌面/墙壁,质心和包围盒全被拉偏。三种情况:

- **YOLO-seg**:模型直接输出实例掩码,拿来即用
- **YOLO-det + SAM2(分支 A1)**:YOLO 的框作为 box prompt 喂 SAM2,在同一张图上生成框内主体的精细掩码。分工:YOLO 决定"是什么、可信度多少",SAM2 决定"哪些像素属于它"
- **YOLO-det 纯几何法(分支 A2)**:不出 2D 掩码,取框内全部像素的 3D 点,在 3D 空间里靠深度差把物体和背景分开(见 [5])

## [3] 域与分辨率对齐(最易出错处,三个子步骤)

### (a) 畸变域 → 去畸变域(仅当 [1] 在原始图上跑)

`undistort.py` 用 `initUndistortRectifyMap` 生成的 map1/map2 语义是"去畸变图的每个像素 ← 原始图的哪个位置"。把原始域二值掩码用**同一套 map** 做 `cv2.remap(mask, map1, map2, cv2.INTER_NEAREST)` 即得去畸变域掩码,再裁同样的 ROI。注意:

- 插值必须**最近邻**——掩码是 0/1 标签,双线性会在边缘产生 0.3、0.7 等无意义值
- **框不能只映射四个角**:畸变下直线是弯的,四角连成的"框"会切掉或多包物体。正确做法:框 → 原始域填充实心矩形掩码 → remap(A1 场景则先出 SAM2 掩码再 remap,更自然)

### (b) 掩码腐蚀 1–2 像素(针对最大误差源)

物体轮廓像素骑在深度不连续处(一侧物体、一侧远处背景),MapAnything 的 pointmap 在这种边缘有平滑,导致轮廓像素的 3D 点"飞"在物体与背景之间的空中(渗色)。`cv2.erode` 向内收 1–2 px,主动放弃最不可靠的边缘一圈。对操作任务损失半厘米轮廓无所谓,质心和包围盒反而更准。

腐蚀放在缩放**前**(全分辨率上做),1–2 px 的物理含义更稳定;最近邻缩放本身还会再吃掉一点边缘。

### (c) 缩放到 pts3d 分辨率

全分辨率掩码最近邻 `cv2.resize` 到 `head_pts3d` 的 H×W(推理 preprocess 定的 518 集分辨率)。此后掩码与 `head_pts3d`、`head_conf`、`head_mask` 逐像素对齐。

## [4] 3D 提升:一次数组索引(方案基石)

```python
valid = obj_mask & head_mask & (head_conf > conf_thr)   # 三重有效性
obj_pts    = head_pts3d[valid]        # N×3, 世界坐标, 米
obj_colors = head_img[valid]          # N×3
obj_conf   = head_conf[valid]         # N
```

传统做法要"深度图 + 内参反投影 + 外参变换"三步;MapAnything 的 pointmap 是**每像素已反投影好的世界坐标**,2D 掩码选中哪些像素就直接拿到哪些 3D 点。全程不需要外参和投影矩阵。`head_mask` 是模型自己的有效像素掩码(天空、无纹理失败区),必须与上。

## [5] 清洗:DBSCAN 取最大簇

腐蚀后掩码仍可能有泄漏(SAM2 把影子算进去、掩码贴到相邻物体),泄漏点在 3D 里是**远离主体的离群小簇**。DBSCAN(`eps` ≈ 2–3 倍体素大小即 3–5 cm,`min_samples` ≈ 10)后保留点数最多的簇。

对分支 A2 这一步是主角:框内所有 3D 点聚类后,深度差天然分出"物体簇"和"背景簇",取**深度最近且框内占比最大**的簇。A2 弱点即在此——物体紧贴背景(平放桌面的薄物体)时两簇粘连。

## [6] 物体建档 → objects.json

```json
{
  "frame": "world = head camera optical center (undistorted)",
  "objects": [{
    "label": "cup", "score": 0.87,
    "centroid": [0.31, 0.12, 0.85],
    "aabb": {"min": [...], "max": [...]},
    "obb":  {"center": [...], "axes": [[...],[...],[...]], "extent": [0.08, 0.08, 0.11]},
    "n_points": 4213, "mean_conf": 1.42
  }]
}
```

- **质心**:3D 点均值(或中值,更抗残余离群点)
- **AABB**:轴对齐包围盒,碰撞粗判
- **OBB**:PCA 主轴 → 物体朝向,抓取规划直接可用(沿短轴合拢手爪)
- ⚠️ 相机只见可见面,点云是"半壳":质心偏向相机一侧、OBB 深度方向偏薄。抓取通常够用;严格场景用已知物体模型配准修正(后话)
- 已知真实尺寸的物体(标准水杯等)的 OBB extent 可反向检验米制尺度 5–15% 误差(PROJECT_LOG §3.3)

## [7] 写入语义体素栅格

任务一体素格已建好(占据/颜色/conf),语义写入:

1. 物体每个点算体素索引 `idx = floor((p - origin) / voxel_size)`
2. 每体素对每类别累积**检测分数加权票数**(落在此格的点数 × YOLO score)
3. 写完后每格取票数最高类别为 `label`,记录归一化置信度
4. 无物体覆盖的占据格 label = 0(背景/未知)

加权投票天然处理:两物体掩码在边界争抢同一体素;以及**多视图融合**——左右手相机 pts3d 在同一世界系,检测结果走相同 [1]–[7] 后往同一栅格投票即可(前提:YOLO 对手腕相机的域也适用,镜头与视角差异不小,需验证)。

最终 `voxels.npz` 每个占据体素带 `(occupied, color, conf, label, label_score)` = **语义占据栅格**:避障用 occupied,找目标用 label。

## [8] 交给机器人:坐标系最后一跳

以上输出均在"head 相机光心 = 原点"的世界系。转基座系只差一个 4×4:

```
p_base = T_base←head @ p_world
```

`T_base←head` 即头部相机手眼标定外参——正好是全配模式要接入的同一份数据,一份标定两处受益。在此之前所有几何量在相机系内自洽,开发验证不受阻塞。
