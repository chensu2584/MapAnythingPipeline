## 2. 重建误差的形状

### 2.1 不是尺度问题

深度诊断拟合 `真实 = a × 模型 + b`：

| 视图 | a | **b** |
|---|---|---|
| head | 0.53 | **+300 mm** |
| hand_left | — | **+440 mm** |
| hand_right | — | **+520 ~ +614 mm** |

`b` 是残差中位数（61–80 mm）的 4–8 倍，**且逐视图不同**。结论：

- **任何全局标量修正都不可能对**。现行 baseline 估计器用三条相机中心距离拟合一个 `s`，给出 RMSE 24→3.7 mm 的"完美拟合"，**对 310 mm 的偏置完全无感**——三个相机之间的距离不含任何场景深度信息。
- **连全局仿射也不成立**：实测仿射拟合内点率仅 **32.6%**（门禁需 ≥0.5），`depth-affine` 模式因此自动回退。

### 2.2 误差分成两类，要分开治

| 类型 | 含义 | 能否用度量深度修 |
|---|---|---|
| **径向**（沿视线） | 物体方向对、距离错 | ✅ 能 |
| **横向**（垂直视线） | 距离对、整体挪到旁边 | ❌ 不能，那是位姿问题 |

实测（5 snapshot 均值，baseline 配置）：

| 视图 | 距离比值 | 径向 | 横向 | 判定 |
|---|---|---|---|---|
| head | 1.147 | +67.5 mm | 32.3 mm | 主要是深度问题 |
| hand_left | 1.387 | +18.5 mm | 28.3 mm | 混合 |
| **hand_right** | 1.075 | +6.3 mm | **36.2 mm** | **主要是位姿问题** |

## 3. 四种干预，只有一种有效

| 干预 | 结果 |
|---|---|
| 全局缩放（baseline） | head 距离比值 1.147（高估 15%） |
| **全局仿射**（实验 1） | ❌ 拟合不上（内点率 32.6%），自动回退到 baseline |
| **遮挡机器人再推理**（实验 2 的 C 组） | ❌ 各项显著变差，见下 |
| **强制标定位姿**（实验 3） | ❌ 变差，且**实验设计有缺陷**，见下 |
| **喂入度量深度** `--depth-input` | ✅ head 1.128→0.976，hand_left 1.373→**1.039** |

`--depth-input` 的证据**不是循环论证**：hand_left 与 hand_right 从未被喂入深度，它们同样改善。

### 3.1 实验 2：把机器人藏起来会让结果更差

正确的对照是 **B vs C**（同样的输出点集，只差模型输入）：

| 视图 | | B（模型能看见） | C（模型看不见） |
|---|---|---|---|
| head | 距离比值 / 横向 | 1.149 / 33.6 mm | **1.215 / 51.3 mm** |
| hand_left | | 1.442 / 102.1 mm | 1.549 / 101.3 mm |
| hand_right | | 1.117 / **36.0 mm** | 1.299 / **78.0 mm** |

相机尺度也从 0.9538 变到 0.9996。**遮挡确实改变了模型的几何估计，但方向是坏的。**

解释：那 40–46% 的像素不是噪声，是**近场结构锚点**；涂成一片平灰不仅拿走信息，大片均匀区域本身还是错误的强信号。

**结论：不要用 `--self-mask-input`。** 但掩膜本身要保留——把机器人的点从**输出**剔除永远是对的。

⚠️ **不要拿 B 和"无掩膜"直接比**：无掩膜时 hand_left 的距离比值 IQR 是 1.055–1.456（双峰），加掩膜后是 1.312–1.463。**被剔除的恰恰是重建最好的那批点**（夹爪在 0.1–0.2 m，近、有纹理）。B 的数字变差是换了统计人群，不是几何变差。

### 3.2 实验 3 的设计缺陷（我方失误）

`calibrated-input` **不是"换成标定位姿"**，它是 README 记录的 legacy hybrid：用标定位姿反投影模型深度，被证实会把场景撕开约 5 cm。所以 exp3 变差是这个已知问题的复现，**不能用来回答位姿假设**。需重做。

## 4. 自遮挡掩膜：过度覆盖

`gripper_*_base_link` 的碰撞几何是 **swept_volume**（夹爪张合全行程包络，16×18×10 cm），距相机仅 ~10 cm，在 113° 视场里投影占 **34% 画面**——远大于实际夹爪。

实测删除率与蓝盒保留（snapshot 0001）：

| | head | hand_left | hand_right |
|---|---|---|---|
| 整体删除 | 25.1% | 36.4% | 43.7% |
| 左蓝盒保留 | 100% | **67.5%** | 59.5% |
| 右蓝盒保留 | 100% | 100% | **0.0%** |

`--shrink-links gripper=0.7`（按质心缩放）的效果：

| shrink | head 掩膜/左盒/右盒 | hand_left | hand_right |
|---|---|---|---|
| 1.00 | 25.7% / 100% / 100% | 39.8% / **22%** / 100% | 46.5% / 70% / **0%** |
| **0.70** | 19.6% / 100% / 100% | 32.1% / **76%** / 100% | 36.3% / **100%** / 11% |
| 0.55 | 17.0% / 100% / 100% | 26.3% / 92% / 100% | 32.6% / 100% / 12% |

**hand_right 看到的右蓝盒即使缩到 0.55 也只剩 12%。**

⚠️ **更正（同日）**：此处最初写成"说明盒子真的在夹爪后面，删除是正确的"。**该结论错误。**
它是从掩膜覆盖率反推的——而掩膜本身正是被质疑的对象，属循环论证。**直接查看 5 张
`hand_right_rgb.png` 原图：两个蓝盒在每一帧都完整、清晰地位于画面中上部，夹爪手指在画面
底部，毫无遮挡。**

真正的原因是：**真实夹爪是镂空结构**——盒子是从手指**之间和旁边**看过去的；而
`swept_volume` 是**实心包络**，把所有缝隙都填上了。因此等比缩放救不了：缩小后它仍是实心块，
照样盖住缝隙里能看到的场景。

### 4.1 URDF 的夹爪描述与实际硬件不符（实测）

操作者指出当前装的不是原装机械手。实测证据：

| | URDF 说 | 实测标定说 |
|---|---|---|
| 左腕相机在末端法兰系的位置 | **(0, 0, 0) mm** | (−2.7, −102.6, +59.4) mm |
| 右腕相机在末端法兰系的位置 | **(0, 0, 0) mm** | (−6.4, +102.6, +64.3) mm |

**URDF 把 `gripper_*_camera_link` 直接放在末端法兰原点上——那是占位符，不是真实相机位置。**
实测值左右镜像对称（Y = ∓102.6 mm，Z ≈ +60 mm），是真实安装。采集脚本把这 118–121 mm 的差
标为"仅供参考"，但它证明 URDF 的末端执行器段至少有一部分不描述实际硬件。

夹爪本体（`swept_volume`）的**位置**大致对得上（叠加后落在可见夹爪上），但**尺寸**是全行程包络。
**它是否描述当前这个夹爪，无法验证。**

⚠️ **因此"改用 `.dae` 手指网格"这条路也不可信**——那些网格描述的可能是另一个末端执行器。

### 4.2 不依赖 URDF 的正解

**夹爪刚性固连在腕相机上 ⇒ 它在图像里的位置永不改变**（时序方差图已证实：右腕相机在 6 个
不同位姿下，夹爪区域始终是低方差区）。

因此**手工在去畸变图上画一次掩膜，对该相机的所有采集永久有效**——不需要 URDF、不需要关节角、
换末端执行器后重画一次即可。入口：`--static-mask hand_right=path.png`。

试过的自动化替代方案及其失败原因（记录以免重复踩）：

| 方法 | 结果 |
|---|---|
| 跨位姿时序方差 | 检出 25.8%，但把**无纹理的白色地面**也算作静态（孔径问题） |
| 时序方差 + 纹理门禁 | 检出 14.8%，但**夹爪自身的白色塑料本体无纹理**被排除，结果破碎 |

`--shrink-links gripper=0.7` 保留为止血手段：它对左盒确实有效（22%→76%），
但对被实心包络填死的缝隙无能为力。

⚠️ **"右蓝盒对齐问题解除"不是修好了对齐**，是掩膜把 hand_right 那份错位的副本删掉了（head 与 hand_left 的两份各保留 100% 且本来就互相对齐）。**问题被掩盖而非解决。**

**采集数据缺夹爪关节角**（22 个关节只有 body/head/arm）。推测与当前装的是夹爪而非灵巧手有关，不同末端执行器的几何与关节定义都不同。因此无法按实际张合角渲染，只能用扫掠体 + 缩放。

## 5. hand_right 对不齐的根因分析

### 5.1 已排除

- ❌ **不是掩膜错位**：用正确内参映射重测，右盒保留 0.0%（比错位版本还低）
- ❌ **不是深度问题**：喂深度后横向误差 43.2 → 40.2 mm，基本不动
- ❌ **不是固定外参错**：横向误差随 capture 变化（13 mm ↔ 43 mm）。按判据"随 capture 变 ⇒ 模型误差，不是固定外参"

### 5.2 实测到的结构性不对称

| 量 | head | hand_left | **hand_right** |
|---|---|---|---|
| **与 head 的光轴夹角** | — | **26–28°** | **39–42°** |
| 与 head 的基线 | — | 0.419–0.426 m | 0.367–0.436 m |
| 与 head 的共视比例 | 100% | 84.1% | 80.7% |
| 有效像素 | 92.7% | 89.1% | **81.0%** |
| depth_max | 2.20 m | 3.81 m | **7.39 m** |
| 视图序号 | 0 | 1 | **2** |

**光轴夹角是最大的不对称：hand_right 始终比 hand_left 偏离 head 多 ~14–16°，在全部 5 个 snapshot 上一致**（尽管手臂姿态在变）。共视比例只差 3.4 个百分点，不足以解释。

### 5.3 根因：三角化随距离变弱（实测推翻了先前两个假设）

先前提出的两个假设（视角过斜 / 视图序号）**都已被数据推翻**：

- **视图序号**：交换 `hand_left`/`hand_right` 喂入顺序后重跑，误差仍跟着**物理相机**（hand_right），
  与序号无关 → 排除 MapAnything 内部机制。
- **视角过斜**：5 个 snapshot 的光轴夹角跨度仅 2.9°，横向误差却变了 4.6×（14→65 mm），
  且相关方向还是反的 → 排除。

**真正的解释**：hand_right 是三台里唯一**观测距离在变**的相机（右臂在动），而它的横向误差与
观测距离强相关：

| snapshot | hand_right 观测中位距 | 横向误差 |
|---|---|---|
| 2128/2129 | 1.07 m | 14–16 mm |
| 2151/2152 | 1.52 m | 42–43 mm |
| 2158 | 1.61 m | 65 mm |

**相机内部 距离↔误差 相关系数 r = +0.950**。折算成角误差是 13 → 41 mrad，**随距离增长而非恒定**，
排除固定朝向误差。误差增速快于距离（距离 ×1.5，误差 ×4.6），符合三角化误差 ∝ d²/B。

head 与手部相机的基线固定 ~0.4 m。场景在 1.07 m 时视差半角 10.6°，1.61 m 时只剩 7.1°。
**视差一弱，这一路位姿约束不住，模型靠先验补，补出来就飘。**

**关键推论：这不是"右手相机的固有缺陷"，是"这台相机被摆得看远处"。** 在最近处（1.07 m）
hand_right 误差 14 mm，是三台里**最好**的（head 31 / hand_left 34）。"总是右手"只是因为
右臂在这批采集里恰好摆得更远。G1 同理。

⚠️ n=5，且相关性来自单台相机的距离变化。坐实需要**故意采一组右臂由近到远的数据**
（0.8/1.2/1.6/2.0 m），看误差是否沿同一曲线。零成本、可证伪。

### 5.4 关键更正：外参正确，是解算问题（决定性证据）

**重投影检验（不依赖 MapAnything）**：把头部深度的 3D 点用 hand_right 的外参 + 内参 + 畸变
投回它的图像——**蓝盒、桌沿、椅子全部精确落位**。左右手都完美。

=> **hand_right 的外参（base_T_camera）几何正确**，一个已知 3D 点投到了正确像素。
配合 FK=0.0000mm、交换顺序误差跟着物理相机走，**三项一起证明：标定/运动学/流水线都没问题，
误差是 MapAnything 的解算输出偏离了正确的输入**。操作者的判断正确。

**G1/G2 都是右手 => 系统性、机械安装决定的不对称**（实测于 G2）：

| | 与 head 光轴夹角(稳定) | 图像 roll vs 重力(随姿态) | 采集脚本外参分解 |
|---|---|---|---|
| hand_left | **26–28°** | 0–8° | 绕 X 转 43° |
| hand_right | **39–42°** | 4–21° | **绕光轴约 180°** |

右手相机比左手多偏离头部 ~14°（5 snapshot 全一致）。头部是锚点(view 0)，右手偏离更大 +
共基线仅 ~0.4m => **视差最弱、共视最小**，三角化约束最松；且图像更倾斜，而 MapAnything 对
重力对齐敏感。若 G1/G2 同款设计（右相机装得更斜/更转），两代都会中招。

**仍未区分**："右手身份本身" vs "更斜的那台相机恰好是右手"。修法不同，用实验区分。

### 5.5 实验 A / B（已实现，GUI 可跑）

- **实验 A — 视图子集**：`--view-order head,hand_right` vs `head,hand_left`，各自只对头部单独重建，
  隔离每台相机的解算质量。GUI: "Views subset"。
- **实验 B — roll 归正**：`--roll-normalize` 把手部图像转正到重力对齐（位姿+内参同步旋转，几何
  自洽，已数值验证颜色一致性 0.9px），测 MapAnything 是否对倾斜相机更差。GUI 勾选框。
  不能与 depth-input/self-mask 同用（后者按未旋转的像素网格对齐）。

判读：跑完对比 `diagnose_reconstruction.py` 的 hand_right `lateral`。
A 里 hand_right 单独也差 => 相机几何本身；B 里归正后变好 => roll/朝向是主因。

### 5.6 应对：根治 vs 规避

**根治方向**（按性价比）：
1. 采集时让手臂离目标更近（1.07 m 误差 14 mm，1.61 m 误差 65 mm，同一台相机）。
2. 增大视差：让头部也动（现在 13 snapshot 头部全静止，浪费一个自由度）；或引入 head stereo/fisheye。
3. 砍远场：hand_right 看到 7–10 m 的办公区，那部分几何弱、拖累位姿估计。
4. 若仍不行，即 MapAnything 低视差下的能力边界 → 只能给该路独立度量约束（腕部深度，暂缓）。

**规避（已实现）**：`--view-max-depth hand_left=1.5 --view-max-depth hand_right=1.5`。
按相机系 z-深度上限剔除各手部相机 1.5 m 外的几何，只保留其可信近场，远场交给 head。
烘进 `views.npz` 的 `{name}_mask`，voxelize/filter_export/scene.ply 全部继承；`pts3d`/`depth_z`
保留全量，远点可恢复。GUI 勾选框默认开、可填数值。

## 6. 当前推荐配置

```bash
# Step A + 掩膜（缩放夹爪扫掠体）
python undistort.py --robot g2 --data-root <session> --output-root <out>
python self_occlusion_mask.py --session <session> --output-root <out> \
    --urdf G2_parameters/G2_t2_crs_omnipicker/urdf/G2_t2_crs_omnipicker.urdf \
    --shrink-links gripper=0.7

# Step B：喂深度，留出 30% 像素保证诊断非循环；不要 --self-mask-input
python run_inference.py --captures <snap> --depth-input --depth-holdout 0.3

# Step C/D：不要 --show_grippers（G1 专用）
python filter_export.py --captures <snap> --max_radius 2.3
python voxelize.py --captures <snap> --voxel_size 0.01 --max_radius 2.3

# 诊断
python diagnose_reconstruction.py --session <session> --output-root <out> --planes
```

**判读 `summary.json` 的三个数**：`residual_rmse_m`（深度传感器自身 3–5 mm）、**`affine_test.b_m`（远大于残差 ⇒ 非纯尺度）**、`inlier_ratio`（<0.5 ⇒ 模型与传感器对不上）。

| 反投影一致性容差 `5e-4` 比 bf16 自身精度（3.9e-3）紧 8 倍 | 远处点误报失败，Step C 中断 |
| `view_bits` 未随过滤掩膜一起筛选 | 带 `--max_radius` 必崩 |
| `world_frame` 无条件写 `"unknown"` | 让 avoidance 侧的回退路径变成硬失败 |
| 掩膜按图像尺寸映射到模型输出 | 预处理先裁剪再缩放，横向拉伸 19%、边缘错位 100 px |
| `self_mask_report or None` 在字典填充前求值 | 实验组在 summary 里看起来像对照组 |
| GUI 行号手工挪动撞格 | Output geometry 下拉被完全遮住 |
| 掩膜只生成不消费 | 实验 2 一度根本没生效 |
| `--view-order` 让 scene.ply 按喂入顺序导出 | filter_export 逐点复核时错位比较，报 4.18 m 偏差 |
